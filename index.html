HTML

  • HTML is one of the three cornerstones of web page creations - foundation and structure.
  • The head element contains information about the webpage.
  • Information in the head element is not visible in the live version of the webpage, except for the title tag.
  • The body element represents the visible content shown to the user.
  • Utilization of different tags within the body element is essential for information categorization along with aesthetic purposes.
  • Visible HTML elements must be contained in between an opening and closing tag.

Git

  • Github is the webinterface of Git, it is a version control software that allows multiple people to make separate changes to a program at the same time.
  • Repositories are projects in Git. It is a folder for your projects that contains all the files that utilizes revision control to safeguard past and future changes.
  • Branch is a workspace where you can make changes that don't affect the live site.
  • Markdown is the way content files are formatted in GitHub. It is a way to write for the web using plain text and easily connects to HTML.
  • Commit changes is to save a new version of a file you are working on. Adds changes to the branch.
  • Pull Request(PR) is when you request for changes to be merged into the main branch to reflect on the live site. It is the best time to discuss and review your work with others.
  • Merge occurs when a PR is approved and pulled into the main branch. Another term is "Deployment".
  • Issues are ways to report new tasks or content fixes. It is mainly used to track progress on the project.
  • Content update workflow: Create a branch (CLI - git checkout) > Add commits (CLI - git commit) > Open a Pull Request (CLI - git pull & git push) > Discuss & Review (GitHub - review content changes) > Merge and Deploy (GitHub - Merge to Main branch)

    • Git commands

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git clone: creates a local copy of your project to local computer
  • git add -A: add all changes through the CLI and commit all files into the working branch
  • git commit -m: commiting changes into the current branch
  • git pull origin main: pull current version of main branch into your local computer prior to pushing changes
  • git push origin feature/branch: push changes into the GitHub repository prior to Pull Request and Merging

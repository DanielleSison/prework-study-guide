HTML

  • HTML is one of the three cornerstones of web page creations - foundation and structure.
  • The head element contains information about the webpage.
  • Information in the head element is not visible in the live version of the webpage, except for the title tag.
  • The body element represents the visible content shown to the user.
  • Utilization of different tags within the body element is essential for information categorization along with aesthetic purposes.
  • Visible HTML elements must be contained in between an opening and closing tag.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Margin and padding follow the CSS Box model.
  • * selector is a special wild card symbol that applies all rules in the external CSS stylsheet to all elements in the HTML.
  • Adding a comma in between two elements in the CSS stylesheet denotes a single rule (DRY - Do not repeat yourself code).
  • There are three ways to style a webpage using CSS: Inline CSS, Internal CSS stylesheet, and External CSS stylesheet.
  • Inline and Internal CSS, are both directly added into the HTML file but are less flexible.
  • External CSS stylesheet is the most common way to style due to its better flexibility.

Git

  • Github is the webinterface of Git, it is a version control software that allows multiple people to make separate changes to a program at the same time.
  • Repositories are projects in Git. It is a folder for your projects that contains all the files that utilizes revision control to safeguard past and future changes.
  • Branch is a workspace where you can make changes that don't affect the live site.
  • Markdown is the way content files are formatted in GitHub. It is a way to write for the web using plain text and easily connects to HTML.
  • Commit changes is to save a new version of a file you are working on. Adds changes to the branch.
  • Pull Request(PR) is when you request for changes to be merged into the main branch to reflect on the live site. It is the best time to discuss and review your work with others.
  • Merge occurs when a PR is approved and pulled into the main branch. Another term is "Deployment".
  • Issues are ways to report new tasks or content fixes. It is mainly used to track progress on the project.
  • Content update workflow: Create a branch (CLI - git checkout) > Add commits (CLI - git commit) > Open a Pull Request (CLI - git pull & git push) > Discuss & Review (GitHub - review content changes) > Merge and Deploy (GitHub - Merge to Main branch)

    • Git commands

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git clone: creates a local copy of your project to local computer
  • git add -A: add all changes through the CLI and commit all files into the working branch
  • git commit -m: commiting changes into the current branch
  • git pull origin main: pull current version of main branch into your local computer prior to pushing changes
  • git push origin feature/branch: push changes into the GitHub repository prior to Pull Request and Merging

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript reads code in a LINEAR way (top to bottom) unless a conditional statement is added.
  • Array is to store a group of data in a variable and use a for loop to perform a task on each piece of data using just a few lines of code.
  • Array is also a single variable that is used to hold a group of data. It is indexed from 0 to n.
  • Scripts should be added near the bottom of an HTML page to allow proper loading.
  • String is a sequence of characters used to represent text.
  • // or /* */ is as an opening or closing command to indicate a comment within the script.
  • The for loop has three parts; starting point, condition statement, and variable shortcut.
  • Function is a fundamental piece of JavaScript language. It is a set of instructions.
  • The two requirements of the JavaScript function are the following; Define the function, and Call the function.
  • Defining the function is also known as function declaration. This must be added in the body of the script.
  • Calling the function is added at the bottom of the script.
